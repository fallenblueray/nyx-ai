### 2. 新增硬截斷機制

**檔案**: `segment/route.ts` 和 `generate-story/route.ts`

```typescript
// 硬截斷：確保不超過 2800 字（給 2500 目標留緩衝）
const MAX_SEGMENT_LENGTH = 2800;
if (segmentText.length > MAX_SEGMENT_LENGTH) {
  // 智能尋找句子結束點進行截斷
  // 若找不到，硬切到 2800 字
}
```

**原理**: 即使模型無視 token 限制，後處理也會強制截斷到合理範圍。

### 3. 上下文注入重構（解決中後段品質問題）

#### 改進前（問題）
```typescript
// 只取結尾 500 字
const previousEnding = previous_segment?.slice(-500) || '';
```

**問題**:
- 結尾可能是高潮片段，導致下一段「重複高潮」
- 丟失了角色初始狀態、場景設定等關鍵信息
- 風格樣本只取結尾，不能代表整體風格

#### 改進後
```typescript
// 取中段 300 字作為風格樣本（更能代表整體風格）
const midPoint = Math.floor(previous_segment.length / 2);
const previousMid = previous_segment?.slice(
  Math.max(0, midPoint - 200),
  midPoint + 200
) || '';

// 取末尾 400 字作為銜接點
const previousEnding = previous_segment?.slice(-400) || '';
```

**新增**: `extractKeyEvents()` 函數，提取關鍵事件序列注入上下文。

## 測試建議

1. **字數測試**: 生成後檢查兩段的字數是否都在 2300-2800 範圍內
2. **風格一致性**: 檢查第二段是否與第一段保持相同語法和描寫風格
3. **劇情連貫性**: 檢查兩段之間是否有明顯斷層或重複
4. **角色一致性**: 確認角色名稱、狀態在兩段中保持一致
